# git-commit

`AI_Context/log.md`와 실제 변경 파일을 교차 분석하여 커밋 메시지를 작성하고 커밋합니다.

## 실행 순서

---

### 1. 변경 파일 확인

아래 두 명령을 실행하여 현재 변경 사항을 파악합니다.

```
git status
git diff --name-only
```

- Untracked / Modified / Deleted 파일 목록을 모두 수집합니다.
- 스테이징 여부와 관계없이 전체 변경 파일을 파악합니다.

---

### 2. log.md 최신 항목 읽기

`AI_Context/log.md` 파일을 읽고 **가장 최근 항목**(상단 첫 번째 `###` 블록)을 확인합니다.

- 해당 항목의 날짜, 작업 제목, 신규·수정·삭제 내역을 파악합니다.
- 이것이 커밋 메시지의 **의도와 맥락** 재료가 됩니다.

---

### 3. 교차 비교 및 커밋 메시지 초안 작성

1단계에서 얻은 실제 변경 파일과 2단계의 log.md 항목을 비교합니다.

- log.md에 기록된 파일과 실제 변경 파일이 일치하는지 확인합니다.
- log.md에 없는 변경 파일이 있다면 별도로 언급합니다.

아래 형식으로 커밋 메시지 초안을 작성합니다.

```
제목: [작업 범위] 한 줄 요약 (50자 이내)

본문:
- 변경 내용 1
- 변경 내용 2
- ...

변경 파일 수: N개
```

- 제목은 log.md의 작업 제목을 기반으로 작성합니다.
- 본문은 log.md의 신규·수정·삭제 항목을 간결하게 요약합니다.
- log.md에 없는 변경 파일은 본문 마지막에 `(log 미기록)` 표시와 함께 추가합니다.

---

### 4. 사용자 확인

작성한 커밋 메시지 초안을 사용자에게 보여주고 확인을 요청합니다.

- 수정 요청이 있으면 메시지를 수정합니다.
- 확인이 되면 다음 단계로 진행합니다.

---

### 5. 커밋 실행

```
git add -A
git commit -m "<확정된 커밋 메시지>"
```

커밋 완료 후 `git log --oneline -1` 로 결과를 확인하여 사용자에게 보여줍니다.

---

## 주의사항

- `AI_Context/log.md`와 실제 변경 파일이 다를 수 있습니다. 실제 파일 기준으로 커밋합니다.
- 커밋은 반드시 사용자 확인 후에만 실행합니다.
- `.gitignore`가 없는 경우 `Library/`, `Temp/`, `obj/` 등 자동 생성 폴더가 포함될 수 있으므로 `git status` 결과를 주의 깊게 검토합니다.
